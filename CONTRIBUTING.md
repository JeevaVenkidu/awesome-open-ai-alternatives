# Contributing

Thank you for improving this list of open-source AI alternatives.

## What Belongs Here

- Projects should be open source, open weights, or clearly labeled when they are proprietary benchmarks.
- Prefer maintained projects with a public repository, documentation, and a clear license.
- Add tools that are useful alternatives to hosted AI products, including ChatGPT alternatives, local inference, agents, RAG, media generation, and observability.
- Do not add referral links, product marketing, duplicates, or projects with no meaningful public activity.

## Entry Format

- Use the existing section and table format.
- Link directly to the canonical GitHub repository or official specification.
- Include a short factual description, current star badge, license, and `Self-Hostable ✅` or `API Only ⚠️` status when relevant.
- Never hard-code star counts; use the shields.io GitHub badge so they remain current.
- Label archived, unreleased, research-only, or license-unclear projects explicitly.

## Verification

- Check that links resolve and that the repository is not archived before submitting.
- Check the repository's latest activity and release notes. Entries with no meaningful update for 12 months should be removed or labeled deprecated.
- Do not present vendor-reported benchmark scores as independent rankings.
- Confirm that model names, context windows, hardware estimates, and licenses match the project's current documentation.

## Pull Requests

1. Fork the repository and create a focused branch.
2. Edit `README.md` and update `CHANGELOG.md` when the change is substantive.
3. Keep the Markdown readable and run the repository link checker locally if possible.
4. Open a pull request explaining what changed and how links and license information were checked.

Please follow the [Code of Conduct](CODE_OF_CONDUCT.md).
